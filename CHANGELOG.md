## [0.1.1] - 2026-07-01
### Added
- 툴바 "+ 그룹"(그룹 생성) 및 "⚙ 설정" 진입 버튼.
- 자동 저장 상태 표시("저장 중…" → "저장됨 hh:mm:ss").
- 그룹↔메모목록, 메모목록↔본문 사이 세로 구분선.
- 색 계열 9종(기본/파랑/청록/초록/노랑/주황/빨강/분홍/보라) × 라이트·다크 팔레트(총 18종), 설정에 색 스와치 선택 UI.

### Changed
- 체크리스트를 "할 일" / "이슈" 두 섹션으로 분리.
- 주간보고를 열 때마다 현재 체크리스트에서 새로 렌더(‘다시 생성’ 없이 자동 최신화). 양식 A↔B 전환 시에도 최신 렌더.
- 시스템 그룹(일일업무일지/주간보고)을 사이드바 하단에 고정 + 가로 구분선으로 분리. 순서변경·일반 메모 이동/생성 금지, 각 타입 전용.
- 테마에서 "강조색 직접 입력"을 제거하고, 색 계열 선택 시 라이트/다크에 맞는 강조색이 자동 적용되도록 단순화(WCAG 대비 검증).
- UI 전반 플랫화: 입력은 테두리 박스 대신 밑줄, 파란 포커스/선택 기본값 제거, 패널 색 한 톤 통일.

### Fixed
- 메모 삭제(🗑)가 목록 드래그 핸들러에 클릭을 빼앗겨 전혀 동작하지 않던 문제(드래그 임계값 + 버튼 제외).
- 선택(열린) 메모 삭제 시 우측 에디터가 비워지지 않던 문제, 편집 중 체크리스트 삭제가 자동저장으로 되살아나던(좀비) 문제.
- 새 메모 생성 시 우측 편집창이 열리지 않던 문제; 제목 편집이 목록에 즉시 반영되지 않던 문제.
- 다크/어두운 테마에서 흰 배경·흰 글씨로 안 보이던 컨트롤 가독성: 콤보 박스·드롭다운, DatePicker 달력 팝업, DataGrid, 우클릭 컨텍스트 메뉴, 대화상자(새 그룹 입력/휴지통 창), 선택 항목 글씨색.
- 고객사 콤보가 선택 시 이름 대신 객체명(ToString)을 표시하던 문제.
- 체크리스트 자동 고객사 분류가 노트를 열거나 자동태깅될 때 "수동 교정"으로 잘못 고정되던 문제.
- 그룹 작업(추가/이름변경/삭제/순서변경) 후 선택 하이라이트가 사라지던 문제; 휴지통 Undo 토스트 자동 해제 및 실행취소 시 복원 그룹으로 이동.

## [0.1.0] - 2026-06-26
### Added
- 일반 메모(plain) 에디터: 디바운스 자동 저장(약 500ms), 저장 버튼 없음.
- 체크리스트(일일 업무일지): 할 일(취소선)/이슈 항목, 고객사 자동 태깅 + 수동 교정.
- 주간보고 자동 생성: 양식 A(할일/이슈 순), 양식 B(고객사별 분류 + 제목줄), 클립보드 복사, 멱등 재생성.
- 고객사 자동 분류 규칙(우선순위 기반, 자율형공장 > SLD), 주차 계산(월~금).
- 그룹(분류) 트리, (미분류) 가상 노드, 시스템 그룹(일일업무일지/주간보고), 그룹 CRUD(ON DELETE SET NULL).
- 전문검색(FTS5): 제목 + 본문 + 체크리스트 항목.
- 휴지통(소프트 삭제) + 복원/영구삭제/Undo, 보존기간 자동 정리(기본 30일).
- 전역 단축키 `Ctrl+Alt+N`(message-only 창), 트레이 상주, 자동시작, 단일 인스턴스(named pipe).
- 테마: 라이트/다크/시스템 모드 + 강조색 + 프리셋 팔레트(모든 색 DynamicResource).
- 데이터 영속화: SQLite WAL, `%LOCALAPPDATA%\Memoria`, 크래시 복구 저널, 자동 백업(VACUUM INTO/Online Backup), 스키마 마이그레이션.
- 문서: README, 아키텍처, 주간보고 양식 규칙, 사용자 가이드(한글).
- CI/CD: GitHub Actions(build+test, 태그 릴리스 단일 exe 자산 첨부).
